Age-Optimal Target Wake Time

Provably Good Wake Schedules for Energy-Constrained Wi-Fi Status Updating
Haoyu Wang, Bo Sheng — University of Massachusetts Boston
Xiaoqian Zhang — University of Nebraska Omaha
Paper (PDF) Technical report (full proofs) Code & data AoI primer

Abstract

Target Wake Time (TWT), introduced in IEEE 802.11ax, lets an access point schedule exactly when each station wakes, transmits, and dozes. Existing TWT schedulers optimize energy or throughput, treating information freshness at best as a constraint and offering no performance guarantees. We design the wake schedule itself for freshness: minimize the weighted average Age of Information (AoI) over stations subject to per-station energy budgets, where the decision variables are the TWT triples (wake interval, offset, service-period duration). We derive a renewal-exact AoI model for TWT under per-SP block fading and validate it against packet-level 802.11ax simulation in ns-3 to within ~1% mean error; we show that, unlike preemptive scheduling, non-preemptive TWT packing can be infeasible at schedule density 1; and we build Harmonic-Greedy, a scheduler with a deterministic 4/ln 2 ≈ 5.77 approximation guarantee under a mild granularity assumption (6/ln 2 ≈ 8.66 unconditionally).

Key result: when does AoI-weighting help?

We compare against a naive baseline (equal-interval) and a strong one (energy-greedy, period-diverse but AoI-unaware), plus a relaxation lower bound. Weighted mean AoI (ms, 10 stations, 10 seeds):

RegimeEqualEnergy-greedy Harmonic-GreedyLB
heterogeneous budget, skew20.919.118.815.8
heterogeneous budget, fade+skew51.345.144.735.7
uniform budget, skew17.217.216.015.2
uniform budget, fade+skew41.641.639.835.9

With heterogeneous budgets, per-station energy floors pin the periods and the strong baseline already matches us. AoI-weighting is isolated in the uniform-budget, binding-density regime, where energy-greedy collapses to equal-interval and only Harmonic-Greedy wins (5–6.5%). With mixed modulation rates the gain reaches 36% at N = 50, decomposing into period diversity (+23%) and AoI-weighting (+17%).

Reproduce

The artifact is an ns-3 contrib module plus a small src/wifi mechanism patch (a TWT power-save manager). On a stock ns-3.48 tree:

git clone https://github.com/Haoyu2/age-optimal-twt
# 1. copy the TWT mechanism into src/wifi and the module into contrib/
cp age-optimal-twt/artifact/src-wifi/twt-power-save-manager.* ns-3/src/wifi/model/
cp -r age-optimal-twt/artifact/ns3-aoi-twt-module ns-3/contrib/aoi-twt
#    (the module's examples/ folder already contains the experiments;
#     register the two src/wifi files in src/wifi/CMakeLists.txt)
# 2. build
cd ns-3 && ./ns3 configure --enable-modules=aoi-twt,wifi,internet,applications \
                          --enable-tests --enable-examples && ./ns3 build
# 3. unit tests, validation, and the full sweep
./test.py -s aoi-twt
python3 age-optimal-twt/artifact/scripts/run_sweep.py 28   # -> results.csv
python3 age-optimal-twt/artifact/scripts/make_macros.py results.csv

See the repository README for full details and the plotting scripts.
